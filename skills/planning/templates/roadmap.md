# Roadmap Template

Save as `.planning/ROADMAP.md`:

```markdown
# Roadmap: [Project Name]

## Overview

[One paragraph describing the journey from start to first release]

## Phases

- [ ] **Phase 1: [Name]** - [One-line description]
- [ ] **Phase 2: [Name]** - [One-line description]
- [ ] **Phase 3: [Name]** - [One-line description]
- [ ] **Phase 4: [Name]** - [One-line description]

## Phase Details

### Phase 1: [Name]
**Goal**: [What this phase delivers as working software]
**Depends on**: Nothing (first phase)

Plans:
- [ ] 01-01: [Description — e.g., "Database schema and migrations"]
- [ ] 01-02: [Description — e.g., "API routes and handlers"]
- [ ] 01-03: [Description — e.g., "Frontend components"]

### Phase 2: [Name]
**Goal**: [What this phase delivers]
**Depends on**: Phase 1

Plans:
- [ ] 02-01: [Description]
- [ ] 02-02: [Description]

## Progress

| Phase | Plans Complete | Status | Completed |
|-------|----------------|--------|-----------|
| 1. [Name] | 0/3 | Not started | - |
| 2. [Name] | 0/2 | Not started | - |
```

<guidelines>
- 3-6 phases total (more = scope creep)
- Each phase delivers coherent, shippable software
- Each plan has 2-3 tasks maximum
- No time estimates
- Status values: `Not started`, `In progress`, `Complete`, `Deferred`
</guidelines>
