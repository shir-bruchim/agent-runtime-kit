---
description: Start a requirements gathering interview to build a project or feature specification
argument-hint: [project-or-feature-name]
---

<objective>
Build a comprehensive specification for "$ARGUMENTS" through guided discovery.
</objective>

<process>
1. Use the spec-interview skill to conduct a structured interview
2. Ask about: problem, users, success criteria, scope, constraints, edge cases
3. Create a spec document at `.planning/SPEC-$ARGUMENTS.md`
4. Offer to create an implementation plan from the spec
</process>

<success_criteria>
Spec document created with clear acceptance criteria, explicit out-of-scope list, and open questions documented.
</success_criteria>
